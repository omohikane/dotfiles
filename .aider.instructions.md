# Chezmoi 管理リポジトリにおける運用ルール

## 1. 作業ディレクトリの厳守

- あなた（aider/Ellie）が作業するのは、実体の `~/.config` ではなく、Chezmoi のソースディレクトリ（`~/.local/share/chezmoi`）です。
- ファイルを新規作成する場合、ディレクトリ名には `dot_` プレフィックスを付けること。
  - 例: `dot_config/nvim/init.lua`

## 2. 自動コミットの禁止

- `commit: false` の設定を尊重し、勝手にコミットしないこと。
- 修正が終わったらプロデューサーが内容を確認し、`chezmoi apply` を経てから手動でコミットします。

## 3. 依存関係と読み込み順（Neovim/dpp.vim）

- `lua/plugins/*.lua` を変更する際は、`dpp.vim` の `depends` が正しいか、最新の仕様に合っているか確認すること。
- 設定エラーで後続（特にカラーテーマ）を止めないよう、`pcall` を使ったガードコードを徹底しなさい。

## 4. 環境変数の扱い

- 個人情報やトークン（`secrets.fish`, `.env` 等）は、Git管理外にするか、Chezmoiのテンプレート機能（`.tmpl`）を使うよう提案すること。

## 5. 破壊的変更前のブランチ作成

- `init.lua` やプラグインマネージャーの根本的な変更を行う際は、必ず「新しいブランチで作業しましょうか？」と確認すること。
